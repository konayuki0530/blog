Hugo+GithubPagesを使って3分でWebサイトを立ち上げる

この記事は TDU Advbent Calendar 2022 22日目の記事です。 初めに 世にはいろいろなブログサービスやCMSやらが溢れていますが、かなりお手軽に個人の静的サイト構築できる方法としてHugo+GithubPagesが挙げられるのではないかと勝手に思っています。 私自身、Hugoに触れたのは最近のことだったのですが、あまりにスムーズに構築できたので感動しました。 ので、今回は慣れていれば3分ほどで出来そうな静的サイト構築の手順を備忘録がわりに記載しておこうと思った次第です。 構築していこう！ はい、構築していきます。 1. リモートリポジトリを作る (30秒) まずはGithubにリモートリポジトリを作っておきましょう。 今回はリポジトリ名を「mysite」にしました。 2. Hugoのインストール　(15秒) 自分の環境にHugoをインストールしましょう。 公式のクイックスタート通りにやれば特に問題ないはず。 $ brew install hugo 自分はbrewで入れました。 3. サイトの作成　(30秒) インストールできたら早速サイトの方を作っていきます。 今回はpagesで公開するのでgithubのユーザー名から以下のような形で生成します。 $ hugo new site [githubのusername].github.io $ cd [githubのusername].github.io $ git init . 4. テーマの導入と設定　(30秒) Hugoにはいろいろなテーマがあるのですが、今回はクイックスタートでも使われている「ananke」を使っていきます git submodule add https://github.com/budparr/gohugo-theme-ananke.git themes/ananke テーマの反映にはconfig.tomlを変更する必要があります。 themeの行を追加し、anankeを指定します。 ついでにbaseURLもリポジトリの名前を追加しておきます theme = "ananke" baseURL = "http://example.org/mysite/" languageCode = "en-us" title = "My New Hugo Site" 5.
